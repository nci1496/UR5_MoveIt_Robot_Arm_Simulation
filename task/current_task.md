# 当前任务卡片

## 任务名称

Phase 6.1 - Qt虚拟画板GUI界面

## 所属阶段

Phase 6 - 虚拟画板系统

## 当前背景

已完成Phase 1-4基础功能（RViz显示、六点控制、椭圆轨迹、8字轨迹）。当前计划开发"虚拟画板"功能，用户在Qt GUI界面绘制任意轨迹，机械臂将跟随绘制的路径移动。

**工作空间定义：**
- 固定平面：y = 0.55
- X范围：-0.4 到 0.3
- Z范围：0.2 到 0.55

**末端执行方式：**
- 使用Marker模拟画笔跟随轨迹
- 无需实际物理画笔装置

## 当前任务目标

设计并实现Qt GUI画板界面，支持用户在界面上自由绘图。

## 本轮优先要做

1. 创建Qt GUI界面基础框架
2. 实现绘图画布（支持鼠标绘制）
3. 实现坐标映射：屏幕像素坐标 -> 机械臂空间坐标
4. 提供"执行"和"清空"按钮

## 当前明确不做

- 轨迹平滑算法（留到下一子任务）
- 轨迹分段/速度控制（留到后续）
- 实时同步执行（先实现录制后播放模式）

## 当前问题定义

Qt GUI如何与ROS节点通信？
- 方案A：Qt GUI与ROS节点分离，通过ROS Topic/Service通信
- 方案B：Qt GUI作为ROS节点的一部分，使用Qt的ROS绑定

建议采用方案A，保持模块独立性。

## 建议实现方向

1. 创建独立Qt项目 `virtual_canvas_gui`
2. 画布尺寸建议：800x400 像素（宽x高）
3. 坐标映射公式：
   - screen_x (0~800) → arm_x (-0.4~0.3)
   - screen_y (0~400) → arm_z (0.2~0.55)
   - screen_z 固定为 0.55
4. 生成路径点后发布到ROS Topic，供机械臂节点订阅执行

## 验收标准

1. Qt窗口能正常显示，包含绘图区域和按钮
2. 用户可以用鼠标在画布上自由绘制曲线
3. 点击"执行"按钮后，画布上显示的轨迹能转换为机械臂坐标并发布
4. "清空"按钮能清除画布内容
