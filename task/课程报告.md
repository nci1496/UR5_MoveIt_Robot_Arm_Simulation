# 基于ROS Noetic与MoveIt的UR5机械臂轨迹规划与控制仿真系统

## 课程报告

---

## 摘要

本报告介绍了一个基于ROS Noetic与MoveIt的UR5机械臂轨迹规划与控制仿真系统的设计与实现。该系统实现了机械臂的多种运动控制功能，包括六点点位控制、笛卡尔直线轨迹运动、椭圆轨迹运动、8字形轨迹运动，以及基于Qt的虚拟画板人机交互系统。系统采用模块化设计，通过ROS话题机制实现GUI与机械臂控制模块的通信，最终实现了"所画即所写"的机械臂书写功能。实验结果表明，该系统能够稳定可靠地完成各类轨迹规划任务，具有良好的可扩展性和可复现性。

**关键词**：ROS Noetic，MoveIt，UR5机械臂，轨迹规划，Qt GUI

## 第1章 绪论

### 1.1 研究背景

随着工业自动化和智能制造的快速发展，工业机器人在生产线上的应用越来越广泛。机械臂作为工业机器人的核心执行机构，其轨迹规划与控制技术是实现精确操作的关键。传统的机械臂控制需要在真实硬件平台上进行调试，这不仅成本高昂，而且存在一定的安全风险。因此，基于仿真的机械臂控制研究具有重要的实际意义。

ROS（Robot Operating System）作为机器人领域的通用软件框架，提供了丰富的工具和库支持机械臂的仿真与控制。MoveIt作为ROS中专门用于机械臂运动规划的功能包，提供了先进的运动规划算法和便捷的编程接口，大大降低了机械臂开发的难度。

### 1.2 研究意义

本项目的研究意义主要体现在以下几个方面：

1. **降低开发成本**：通过仿真环境进行机械臂控制算法的开发和测试，无需昂贵的硬件设备即可完成大部分开发工作。

2. **提高安全性**：在仿真环境中进行实验，可以有效避免因程序错误导致的设备损坏和安全事故。

3. **便于教学演示**：仿真系统可以直观地展示机械臂的运动过程，便于学生理解和学习机械臂控制原理。

4. **促进创新**：模块化的设计使得新功能的添加和算法的改进变得更加容易。

### 1.3 主要工作内容

本项目的主要工作内容包括：

- 在ROS Noetic平台上搭建UR5机械臂仿真环境
- 实现六点点位控制功能
- 实现笛卡尔直线轨迹规划
- 实现椭圆轨迹和8字形轨迹规划
- 开发基于Qt的虚拟画板GUI系统
- 实现GUI与机械臂控制的ROS通信集成

## 第3章 机器人轨迹规划设计

### 3.1 六点点位控制

#### 3.1.1 功能描述

六点点位控制是最基础的机械臂运动控制方式，通过依次向机械臂发送六个目标位姿，使机械臂按顺序到达各个空间点位。该功能验证了机械臂点位运动的基本能力。

#### 3.1.2 核心代码实现

```cpp
// 初始化MoveIt接口
ros::init(argc, argv, "six_points");
ros::AsyncSpinner spinner(1);
spinner.start();

moveit::planning_interface::MoveGroupInterface move_group("manipulator");

// 定义6个目标位姿
geometry_msgs::Pose target_pose[6];

// 初始化姿态（保持末端方向一致）
for(int i = 0; i < 6; i++) {
    target_pose[i].orientation.w = 1.0;
}

// 设置6个目标点位坐标
target_pose[0].position = {0.40,  0.20,  0.40};
target_pose[1].position = {0.40, -0.20,  0.40};
target_pose[2].position = {0.55,  0.00,  0.30};
target_pose[3].position = {0.40, -0.20,  0.40};
target_pose[4].position = {0.40,  0.20, -0.40};
target_pose[5].position = {-0.40, 0.20,  0.40};

// 依次移动到各点位
for(int i = 0; i < 6; i++) {
    move_group.setPoseTarget(target_pose[i]);

    bool success = (move_group.move() ==
                    moveit::core::MoveItErrorCode::SUCCESS);

    if(success)
        ROS_INFO("Move to point %d Success!", i+1);
    else
        ROS_WARN("Move to point %d Failed!", i+1);
}
```

**关键函数说明**：

| 函数 | 说明 |
|------|------|
| `MoveGroupInterface("manipulator")` | 创建机械臂控制接口，"manipulator"是UR5的规划组名称 |
| `setPoseTarget(pose)` | 设置目标位姿，包含位置(position)和姿态(orientation) |
| `move()` | 执行运动规划，返回SUCCESS表示规划成功 |

### 3.2 笛卡尔直线轨迹

#### 3.2.1 功能描述

笛卡尔直线轨迹规划使用MoveIt的computeCartesianPath函数，实现机械臂末端在笛卡尔空间沿直线运动。该功能保证了机械臂末端执行器在两点之间沿直线路径移动，避免奇异构型。

#### 3.2.2 算法原理

笛卡尔路径规划的核心思想是将期望轨迹离散为多个路径点，然后通过插值生成平滑的运动路径。

```
起点 ──────────────────────────── 终点
  │          │          │          │
  ▼          ▼          ▼          ▼
[Pose0] → [Pose1] → [Pose2] → [Pose3]
   │          │          │          │
   └──────────┴──────────┴──────────┘
            computeCartesianPath()
                    │
                    ▼
            插值平滑 + 运动学求解
                    │
                    ▼
            RobotTrajectory
```

**computeCartesianPath函数参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| waypoints | vector<Pose> | 路径点序列 |
| eef_step | double | 末端步进距离(m)，值越小轨迹越平滑 |
| jump_threshold | double | 跳跃阈值，防止奇异构型，0表示禁用 |
| trajectory | RobotTrajectory& | 输出的规划轨迹 |

#### 3.2.3 核心代码实现

```cpp
// 获取起始位姿
geometry_msgs::Pose start_pose = move_group.getCurrentPose().pose;
geometry_msgs::Pose end_pose = start_pose;
end_pose.position.x += 0.3;  // 沿X轴移动30cm

// 构建路径点
std::vector<geometry_msgs::Pose> waypoints;
waypoints.push_back(start_pose);
waypoints.push_back(end_pose);

// 笛卡尔路径规划
moveit_msgs::RobotTrajectory trajectory;
double fraction = move_group.computeCartesianPath(
    waypoints,      // 路径点序列
    0.01,           // eef_step: 0.01m步进
    0.0,            // jump_threshold: 禁用
    trajectory);

// fraction表示路径覆盖率，>0.95表示规划成功
if(fraction > 0.95) {
    // 创建运动计划并执行
    moveit::planning_interface::MoveGroupInterface::Plan plan;
    plan.trajectory_ = trajectory;
    move_group.execute(plan);
}
```

### 3.3 椭圆轨迹规划

#### 3.3.1 功能描述

椭圆轨迹规划通过数学方程生成椭圆曲线路径点，使机械臂末端沿椭圆轨迹运动。该功能展示了机械臂绘制复杂曲线的能力。

#### 3.3.2 数学模型

椭圆轨迹的参数方程如下：

$$x = a \cdot \cos(t)$$
$$y = b \cdot \sin(t)$$
$$z = 0$$

其中：
- $a = 0.40m$（长半轴）
- $b = 0.30m$（短半轴）
- $t \in [0, 2\pi]$，步进0.1弧度

#### 3.3.3 核心代码实现

```cpp
// 发布Marker用于可视化轨迹
ros::Publisher marker_pub = nh.advertise<visualization_msgs::Marker>(
    "trajectory_marker", 1, true);

visualization_msgs::Marker line_strip;
line_strip.header.frame_id = "base_link";
line_strip.type = visualization_msgs::Marker::LINE_STRIP;
line_strip.scale.x = 0.01;     // 线宽1cm
line_strip.color.r = 1.0;      // 红色
line_strip.color.a = 1.0;      // 完全不透明

// 椭圆参数
double a = 0.40;  // 长半轴
double b = 0.30;  // 短半轴

geometry_msgs::Pose start_pose = move_group.getCurrentPose().pose;
std::vector<geometry_msgs::Pose> waypoints;

// 生成椭圆轨迹点
for(double t = 0; t <= 2*M_PI; t += 0.1) {
    geometry_msgs::Pose pose;
    pose.position.x = a * cos(t);
    pose.position.y = b * sin(t);
    pose.position.z = 0;
    pose.orientation.w = start_pose.orientation.w;  // 保持末端姿态

    waypoints.push_back(pose);

    // 添加Marker点用于可视化
    geometry_msgs::Point p;
    p.x = pose.position.x;
    p.y = pose.position.y;
    p.z = pose.position.z;
    line_strip.points.push_back(p);
}

// 笛卡尔路径规划
moveit_msgs::RobotTrajectory trajectory;
double fraction = move_group.computeCartesianPath(
    waypoints, 0.01, 0.0, trajectory);

ROS_INFO("Fraction = %.2f", fraction);

// 成功率>95%时执行
if(fraction > 0.95) {
    marker_pub.publish(line_strip);  // 发布Marker可视化
    ros::Duration(1.0).sleep();     // 等待Marker显示

    move_group.execute(plan);        // 执行轨迹
}
```

### 3.4 8字形轨迹规划

#### 3.4.1 功能描述

8字形轨迹是更具挑战性的轨迹规划任务，其数学模型是两个椭圆在垂直方向上的叠加。该功能进一步验证了机械臂绘制复杂三维轨迹的能力。

#### 3.4.2 数学模型

8字轨迹的参数方程如下：

$$x = 0.4$$
$$y = b \cdot \sin(2t)$$
$$z = a \cdot \cos(t)$$

关键在于Y坐标使用 $\sin(2t)$，产生两倍频率，从而形成8字形：

```
    z
    ↑
    │   ╭───╮
    │  ╱     ╲
    │ ╱   ╳   ╲    ← 交叉点
    │ ╲   ╳   ╱
    │  ╲     ╱
    │   ╰───╯
    └──────────→ y
```

#### 3.4.3 核心代码实现

```cpp
// 8字轨迹参数
double a = 0.40;  // 长半轴（控制Z方向幅度）
double b = 0.30;  // 短半轴（控制Y方向幅度）

std::vector<geometry_msgs::Pose> waypoints;

for(double t = 0; t <= 2*M_PI; t += 0.1) {
    geometry_msgs::Pose pose;
    pose.position.x = 0.4;              // X保持固定
    pose.position.y = b * sin(2*t);     // 2倍频率，形成8字
    pose.position.z = a * cos(t);       // Z方向振荡
    pose.orientation.w = 1.0;

    waypoints.push_back(pose);
}

// 笛卡尔路径规划
double fraction = move_group.computeCartesianPath(
    waypoints, 0.01, 0.0, trajectory);
```

## 第4章 虚拟画板系统

### 4.1 系统总体架构

虚拟画板系统是本项目的创新功能，实现了"所画即所写"的机械臂书写功能。系统由三个主要模块组成：

```
+------------------+       +---------------------+       +-------------+
|  Qt虚拟画板GUI   | ---> | /canvas_trajectory  | ---> | canvas_     |
| (virtual_canvas) | 发布  | (geometry_msgs/     | 订阅  | executor    |
|                  |       |  PoseArray)         |       |             |
+------------------+       +---------------------+       +------+------+
                                                                  |
                                                                  v
                                                         +-------------+
                                                         | UR5机械臂   |
                                                         | (RViz仿真)  |
                                                         +-------------+
```

**数据流向**：
1. 用户在Qt画板GUI中用鼠标绘制图形
2. GUI将屏幕坐标转换为机械臂坐标
3. 通过ROS话题 `/canvas_trajectory` 发布轨迹
4. canvas_executor订阅话题并控制机械臂执行

### 4.2 Qt GUI模块

#### 4.2.1 功能特性

#### 4.2.2 坐标映射算法

工作空间定义：
- 固定平面：y = 0.55
- X范围：-0.4 到 0.3
- Z范围：0.2 到 0.55

```cpp
QPointF screenToArm(const QPoint &screenPos) const {
    // 归一化屏幕坐标
    double normalized_x = screenPos.x() / coordMapping.screen_width;
    double normalized_y = screenPos.y() / coordMapping.screen_height;

    // 镜像处理
    if (mirror_) {
        normalized_x = 1.0 - normalized_x;
    }
    if (mirrorZ_) {
        normalized_y = 1.0 - normalized_y;
    }

    // 映射到机械臂坐标
    double arm_x = coordMapping.arm_x_min +
                   normalized_x * (coordMapping.arm_x_max - coordMapping.arm_x_min);

    double arm_z = coordMapping.arm_z_min +
                   normalized_y * (coordMapping.arm_z_max - coordMapping.arm_z_min);

    return QPointF(arm_x, arm_z);
}
```

### 4.4 canvas_executor执行器

#### 4.4.1 核心代码实现

```cpp
class CanvasExecutor {
public:
    CanvasExecutor()
        : move_group_("manipulator")
    {
        // 订阅画板轨迹话题
        trajectory_sub_ = nh_.subscribe(
            "/canvas_trajectory", 10,
            &CanvasExecutor::trajectoryCallback, this);

        // 发布Marker用于RViz可视化
        marker_pub_ = nh_.advertise<visualization_msgs::Marker>(
            "trajectory_marker", 10, true);
    }

    void trajectoryCallback(const geometry_msgs::PoseArray::ConstPtr& msg) {
        if (msg->poses.empty()) return;

        ROS_INFO("Received trajectory with %zu points", msg->poses.size());

        // 发布Marker可视化
        publishTrajectoryMarker(msg);

        // 执行笛卡尔轨迹
        executeCartesianTrajectory(msg);
    }

    void executeCartesianTrajectory(const geometry_msgs::PoseArray::ConstPtr& trajectory) {
        std::vector<geometry_msgs::Pose> waypoints;

        // 从消息构建路径点
        for (const auto& pose_msg : trajectory->poses) {
            waypoints.push_back(pose_msg);
        }

        // 笛卡尔路径规划
        moveit_msgs::RobotTrajectory trajectory_msg;
        double fraction = move_group_.computeCartesianPath(
            waypoints,
            0.01,      // eef_step: 10mm
            0.0,       // jump_threshold
            trajectory_msg,
            true);     // avoid_collisions

        ROS_INFO("Cartesian path: %.1f%%", fraction * 100);

        // 执行轨迹
        if (fraction > 0.5) {
            move_group_.execute(trajectory_msg);
        }
    }
};
```
