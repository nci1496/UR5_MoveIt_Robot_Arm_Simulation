# 基于ROS Noetic与MoveIt的UR5机械臂轨迹规划与控制仿真系统

## 课程报告

---

## 摘要

本报告介绍了一个基于ROS Noetic与MoveIt的UR5机械臂轨迹规划与控制仿真系统的设计与实现。该系统实现了机械臂的多种运动控制功能，包括六点点位控制、笛卡尔直线轨迹运动、椭圆轨迹运动、8字形轨迹运动，以及基于Qt的虚拟画板人机交互系统。系统采用模块化设计，通过ROS话题机制实现GUI与机械臂控制模块的通信，最终实现了"所画即所写"的机械臂书写功能。实验结果表明，该系统能够稳定可靠地完成各类轨迹规划任务，具有良好的可扩展性和可复现性。

**关键词**：ROS Noetic，MoveIt，UR5机械臂，轨迹规划，Qt GUI

## 第1章 绪论

### 1.1 研究背景

随着工业自动化和智能制造的快速发展，工业机器人在生产线上的应用越来越广泛。机械臂作为工业机器人的核心执行机构，其轨迹规划与控制技术是实现精确操作的关键。传统的机械臂控制需要在真实硬件平台上进行调试，这不仅成本高昂，而且存在一定的安全风险。因此，基于仿真的机械臂控制研究具有重要的实际意义。

ROS（Robot Operating System）作为机器人领域的通用软件框架，提供了丰富的工具和库支持机械臂的仿真与控制。MoveIt作为ROS中专门用于机械臂运动规划的功能包，提供了先进的运动规划算法和便捷的编程接口，大大降低了机械臂开发的难度。

### 1.2 研究意义

本项目的研究意义主要体现在以下几个方面：

1. **降低开发成本**：通过仿真环境进行机械臂控制算法的开发和测试，无需昂贵的硬件设备即可完成大部分开发工作。

2. **提高安全性**：在仿真环境中进行实验，可以有效避免因程序错误导致的设备损坏和安全事故。

3. **便于教学演示**：仿真系统可以直观地展示机械臂的运动过程，便于学生理解和学习机械臂控制原理。

4. **促进创新**：模块化的设计使得新功能的添加和算法的改进变得更加容易。

### 1.3 主要工作内容

本项目的主要工作内容包括：

- 在ROS Noetic平台上搭建UR5机械臂仿真环境
- 实现六点点位控制功能
- 实现笛卡尔直线轨迹规划
- 实现椭圆轨迹和8字形轨迹规划
- 开发基于Qt的虚拟画板GUI系统
- 实现GUI与机械臂控制的ROS通信集成

## 第3章 机器人轨迹规划设计

### 3.1 六点点位控制

#### 3.1.1 功能描述

六点点位控制是最基础的机械臂运动控制方式，通过依次向机械臂发送六个目标位姿，使机械臂按顺序到达各个空间点位。该功能验证了机械臂点位运动的基本能力。

#### 3.1.2 目标点位设置

六个目标点位的坐标设置如下：

| 点位 | X (m) | Y (m) | Z (m) |
|------|-------|-------|-------|
| 点1 | 0.40 | 0.20 | 0.40 |
| 点2 | 0.40 | -0.20 | 0.40 |
| 点3 | 0.55 | 0.00 | 0.30 |
| 点4 | 0.40 | -0.20 | 0.40 |
| 点5 | 0.40 | 0.20 | -0.40 |
| 点6 | -0.40 | 0.20 | 0.40 |

所有点位的末端姿态保持一致（orientation.w = 1.0），仅改变位置信息。

#### 3.1.3 运行指令

```bash
rosrun arm_control_demo six_points
```

### 3.2 笛卡尔直线轨迹

#### 3.2.1 功能描述

笛卡尔直线轨迹规划使用MoveIt的computeCartesianPath函数，实现机械臂末端在笛卡尔空间沿直线运动。该功能保证了机械臂末端执行器在两点之间沿直线路径移动。

#### 3.2.2 运行指令

```bash
rosrun arm_control_demo cartesian_line
```

![图3-2 笛卡尔直线轨迹执行效果](report_images/fig3-2-cartesian.png)
**图3-2 笛卡尔直线轨迹执行效果**

### 3.3 椭圆轨迹规划

#### 3.3.1 功能描述

椭圆轨迹规划通过数学方程生成椭圆曲线路径点，使机械臂末端沿椭圆轨迹运动。该功能展示了机械臂绘制复杂曲线的能力。

#### 3.3.2 数学模型

椭圆轨迹的参数方程如下：

$$x = a \cdot \cos(t)$$
$$y = b \cdot \sin(t)$$
$$z = 0$$

其中，$a = 0.40m$（长半轴），$b = 0.30m$（短半轴），$t \in [0, 2\pi]$

#### 3.3.3 运行指令

```bash
rosrun arm_control_demo ellipse_path
```

### 3.4 8字形轨迹规划

#### 3.4.1 功能描述

8字形轨迹是更具挑战性的轨迹规划任务，其数学模型是两个椭圆在垂直方向上的叠加。该功能进一步验证了机械臂绘制复杂三维轨迹的能力。

#### 3.4.2 数学模型

8字轨迹的参数方程如下：

$$x = 0.4$$
$$y = b \cdot \sin(2t)$$
$$z = a \cdot \cos(t)$$

#### 3.4.3 运行指令

```bash
rosrun arm_control_demo eight_path
```

## 第4章 虚拟画板系统

### 4.1 系统总体架构

虚拟画板系统是本项目的创新功能，实现了"所画即所写"的机械臂书写功能。系统由三个主要模块组成：

```
+------------------+       +---------------------+       +-------------+
|  Qt虚拟画板GUI   | ---> | /canvas_trajectory  | ---> | canvas_     |
| (virtual_canvas) | 发布  | (geometry_msgs/     | 订阅  | executor    |
|                  |       |  PoseArray)         |       |             |
+------------------+       +---------------------+       +------+------+
                                                                  |
                                                                  v
                                                         +-------------+
                                                         | UR5机械臂   |
                                                         | (RViz仿真)  |
                                                         +-------------+
```

### 4.2 Qt GUI模块

#### 4.2.1 功能特性

Qt虚拟画板GUI提供以下功能：
- 鼠标自由绘图
- 屏幕坐标到机械臂坐标的映射
- 轨迹平滑处理
- 速度控制（1%-100%）
- 采样率调节（1-20像素）
- X轴和Z轴镜像控制
- 执行/清空按钮

![图4-1 Qt GUI界面](report_images/fig4-1-qt-gui.png)
**图4-1 Qt GUI界面**

![图4-2 GUI中绘制图形的界面](report_images/fig4-2-drawing.png)
**图4-2 GUI中绘制图形的界面**

#### 4.2.2 坐标映射

工作空间定义：
- 固定平面：y = 0.55
- X范围：-0.4 到 0.3
- Z范围：0.2 到 0.55

### 4.3 ROS通信模块

GUI通过ROS话题发布轨迹数据：

| 话题名 | 类型 | 方向 | 说明 |
|--------|------|------|------|
| /canvas_trajectory | geometry_msgs/PoseArray | GUI → Executor | 发布绘制轨迹 |
| /trajectory_marker | visualization_msgs/Marker | Executor → RViz | 显示轨迹预览 |

### 4.4 完整运行流程

```bash
# 终端1：启动机械臂仿真环境
roslaunch ur5_moveit_config demo.launch

# 终端2：启动Qt GUI
rosrun virtual_canvas_gui virtual_canvas_gui

# 终端3：启动执行器
rosrun canvas_executor canvas_executor
```
