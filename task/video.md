## 场景6：椭圆轨迹规划

### 6.1 原理说明

**画面**：数学公式

**配音**：

> 椭圆轨迹规划通过数学方程生成路径点。
> 我们使用参数方程：X等于长半轴乘以cos t，Y等于短半轴乘以sin t。
> 其中长半轴为0.4米，短半轴为0.3米。

### 6.2 运行演示

**画面**：终端 + RViz

**配音**：

> 现在运行椭圆轨迹程序。

**操作**：
```bash
rosrun arm_control_demo ellipse_path
```

**配音**：

> 可以看到，终端显示"Fraction = 1.00"，表示路径规划100%成功。
> 同时，红色Marker显示的椭圆轨迹预览与实际运动轨迹完全吻合。
> 机械臂末端沿椭圆曲线平滑运动。

**画面**：机械臂绘制椭圆的过程

## 场景7：8字形轨迹规划

### 7.1 原理说明

**画面**：数学公式

**配音**：

> 8字形轨迹是一个更具挑战性的任务。
> 它实际上是两个椭圆在垂直方向上的叠加。
> 我们使用参数方程：X保持为0.4，Y等于短半轴乘以sin 2t，Z等于长半轴乘以cos t。
> 注意这里Y坐标使用了2倍频率，这就是形成8字的关键。

### 7.2 运行演示

**画面**：终端 + RViz

**配音**：

> 现在运行8字轨迹程序。

**操作**：
```bash
rosrun arm_control_demo eight_path
```

**配音**：

> 机械臂末端沿8字形轨迹运动。
> 从Marker预览可以看到完整的8字形状。
> 这个轨迹在三维空间中完成，充分展示了机械臂的运动控制能力。

## 场景8：虚拟画板系统

### 8.1 系统架构

**画面**：架构图

**配音**：

> 最后，我为大家介绍本项目的创新功能：虚拟画板系统。
> 这个系统实现了"所画即所写"的书写功能。
> 系统由三个模块组成：Qt GUI画板负责绘图和坐标映射；ROS话题负责数据传输；canvas_executor执行器负责控制机械臂。

### 8.2 启动系统

**画面**：三个终端窗口

**配音**：

> 首先启动机械臂仿真环境。

**操作1**：
```bash
# 终端1
roslaunch ur5_moveit_config demo.launch
```

**配音**：

> 然后启动Qt虚拟画板GUI。

**操作2**：
```bash
# 终端2
rosrun virtual_canvas_gui virtual_canvas_gui
```

**配音**：

> 最后启动执行器节点。

**操作3**：
```bash
# 终端3
rosrun canvas_executor canvas_executor
```

### 8.3 GUI界面介绍

**画面**：Qt GUI界面

**配音**：

> 这是Qt虚拟画板的主界面。
> 左侧是绘图区域，我们可以用鼠标自由绘制。
> 右侧是控制面板，包括：
> - 速度滑块：可以调节机械臂运动速度，从1%到100%；
> - 采样率滑块：调节轨迹采样精度；
> - 镜像控制：可以开关X轴和Z轴镜像，方便调整书写方向；
> - 执行按钮：点击后发送轨迹给机械臂；
> - 清空按钮：清除画布内容。

### 8.4 功能演示

**画面**：GUI + RViz双屏

**配音**：

> 现在，我在画板上画一个简单的图形，比如一个字母L。

**操作**：鼠标在画板上绘制

**配音**：

> 绘制完成后，点击"执行"按钮。

**操作**：点击执行

**配音**：

> 可以看到终端显示"Trajectory published to /canvas_trajectory"，说明轨迹已成功发布。
> 同时，RViz中显示了红色Marker轨迹预览。
> 现在，机械臂开始执行轨迹！

**画面**：机械臂书写L

**配音**：

> 机械臂末端按照我们绘制的轨迹运动，完美复现了L形图案。
> 这就是"所画即所写"的效果。

### 8.5 更多功能展示

**画面**：GUI界面

**配音**：

> 我们再来试试更复杂的图形。

## 录制注意事项

1. **光线**：确保录制环境光线充足，屏幕清晰可见
2. **分辨率**：建议使用1920x1080或更高分辨率
3. **鼠标**：录制时鼠标指针要清晰，必要时可以高亮显示
4. **速度**：操作演示时语速适中，留给观众理解时间
5. **切换**：场景切换时可以使用淡入淡出效果
6. **备份**：录制前务必保存所有工作
