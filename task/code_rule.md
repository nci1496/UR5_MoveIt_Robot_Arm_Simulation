# 代码规则

## 总原则

每次只完成一个小目标。

完成后必须等待人工验收。

不得一次性生成大量文件。

不得未经确认重构项目。

每次写完后，不需要编译，给我运行指令和验收标准即可，我会反馈。
---

## 文件管理

新增功能使用独立cpp文件。

禁止修改已稳定运行文件。

例如：

six_points.cpp

ellipse_path.cpp

eight_path.cpp

均视为已验收文件。

---

## 开发流程

1. 提出方案

2. 说明原因

3. 生成代码

4. 编译验证

5. 人工验收

6. 更新TaskHistory

---

## 禁止事项

禁止一次生成20个以上文件。

禁止生成无法验证的大型框架。

禁止跳过验收步骤。
